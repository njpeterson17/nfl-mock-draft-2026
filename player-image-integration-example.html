Player Image System Integration Guide

This guide shows how to integrate the new Player Image System into the existing NFL Mock Draft website.

Step 1: Add CSS Files

Add these lines to the <head> section of index.html, after the existing CSS imports:

<!-- Player Image System Styles -->
<link rel="stylesheet" href="css/player-images.css">
<link rel="stylesheet" href="css/player-initials.css">

Step 2: Add JavaScript Files

Add these lines before the closing </body> tag in index.html:

<!-- Player Image System -->
<script src="js/player-image-database.js"></script>
<script src="js/player-image-loader.js"></script>
<script src="js/player-image-integration.js"></script>
<script src="js/index-image-integration.js"></script>

Step 3: Update Pick Cards (Optional)

The integration script will automatically upgrade existing pick cards. However, for best results, update your pick cards to use data attributes:

Before:

<div class="player-photo">
    <img src="images/players/francis-mauigoa.jpg" alt="Francis Mauigoa">
</div>

After:

<div class="player-photo"
     data-player-name="Francis Mauigoa"
     data-player-position="OT"
     data-player-school="Miami">
</div>

Step 4: Manual JavaScript Usage

For dynamic content or custom integrations:

// Load image into container by ID
loadPlayerImage('photoContainer', 'Fernando Mendoza', 'QB', 'Indiana');

// Create new player photo element
const photo = createPlayerPhoto({
    name: 'Caleb Downs',
    position: 'S',
    school: 'Ohio State'
}, { size: 'large' });

// Refresh all images on page
refreshPlayerImages();

// Access database
const data = PlayerImageDatabase.get('Fernando Mendoza');
const stats = PlayerImageDatabase.getStats();

// Direct loader access
await playerImageLoader.load(container, name, position, school);

Features

  • Lazy Loading: Images load only when scrolled into view
  • Smart Fallbacks: Local → ESPN → CBS → Team Logo → Initials
  • Caching: Loaded images cached for performance
  • Beautiful Placeholders: CSS-based initials when no image available
  • Team Colors: Placeholders use school colors
  • Position Colors: Each position has a unique color scheme
  • Auto-Integration: Works with existing HTML automatically
